# 要求定義書

> 「何を作るか・なぜ作るか」を定義する。技術スタックの詳細は書かない。

---

## 1. プロジェクト概要

アプリケーションログの記録・検索・分析を一元化する Web アプリケーション。ログデータから意味のある指標を可視化することで、システムの状態把握・問題の早期発見・意思決定を支援する。

詳細なペルソナ・ユースケース定義は [`docs/requirements/personas_usecases.md`](./personas_usecases.md) を参照。

---

## 3. 機能要求

### 3.1 ログ管理（必須）

| # | 要求 |
|---|------|
| F-01 | ログを新規作成できる |
| F-02 | ログ一覧を表示できる |
| F-03 | ログの詳細を表示できる |
| F-04 | ログを編集できる |
| F-05 | ログを削除できる（管理者のみ） |

### 3.2 検索・フィルタ（必須）

| # | 要求 |
|---|------|
| F-06 | 日付範囲でログを絞り込める |
| F-07 | severity（重大度）でログを絞り込める |
| F-08 | source（発生源）でログを絞り込める |
| F-09 | ログ一覧をソートできる |
| F-10 | ログ一覧をページネーションで表示できる |

### 3.3 分析ダッシュボード（必須）

| # | 要求 |
|---|------|
| F-11 | severity 別サマリーカードで ERROR / CRITICAL 件数を確認できる |
| F-12 | 日付範囲・severity・source を組み合わせてログを集計できる |
| F-13 | severity または source ごとの時系列ログ件数トレンドをチャートで表示できる |
| F-14 | フィルタパネルで集計条件を動的に変更できる |
| F-15 | 手動更新ボタンで最新データを取得できる |

### 3.4 ボーナス機能（任意）

| # | 要求 |
|---|------|
| F-16 | ログデータを CSV ファイルとしてダウンロードできる |
| F-17 | 日付範囲・source ごとの severity 分布をヒストグラムで表示できる |
| F-18 | ユニットテストまたはインテグレーションテストが実装されている |

---

## 4. データ要求

ログデータは以下の属性を持つ。

| 属性 | 説明 |
|------|------|
| `timestamp` | ログが発生した日時 |
| `message` | ログのメッセージ本文 |
| `severity` | ログの重大度（INFO / WARNING / ERROR / CRITICAL） |
| `source` | ログの発生源（サービス名・モジュール名など任意の文字列） |

---

## 5. 非機能要求

| # | 要求 | 区分 |
|---|------|------|
| NF-01 | 入力値のバリデーションを実施し、不正なデータを受け付けない | 品質 |
| NF-02 | API・UI のエラーを適切にハンドリングし、ユーザーにわかりやすいメッセージを表示する | 品質 |
| NF-03 | サーバーサイドでのロギングを実装する | 品質 |
| NF-04 | docker-compose 一発で開発環境を起動できる | 運用性 |
| NF-05 | README にアプリケーションの起動方法・設計思想・技術選定理由を記載する | 保守性 |
