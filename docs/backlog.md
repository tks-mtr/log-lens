# バックログ（やりたかったこと）

---

## 認証・権限管理

- **やりたかったこと:** JWT ベースの認証を実装し、管理者／一般ユーザーのロールを実際に分離する
- **理由:** 現状は画面を管理者ロール前提で構築しており、削除などの操作に制限がない
- **実現方法案:** FastAPI の `OAuth2PasswordBearer` + JWT + ユーザーテーブルの追加

---

## リテンションポリシー（自動削除）

- **やりたかったこと:** 一定期間（例: 90日）経過したログを自動削除する仕組みを実装する
- **理由:** ログは蓄積し続けるとストレージを圧迫するため、運用上は保持期間の管理が必要
- **実現方法案:** PostgreSQL の `pg_cron` 拡張 or バックグラウンドジョブ（APScheduler 等）

---

## AI ディベートエンジンによるインサイト抽出

- **やりたかったこと:** OSS の LLM を2台並列で動作させ、ログデータ・要求定義・要件定義を入力として、2つのモデルがディベート形式で「何が問題か・どこに価値があるか」を随時分析・提示する機能を実装する
- **理由:** 大量のログから人間が価値を見出すには限界がある。LLM 同士の対立的な議論（ディベート）を通じることで、単一モデルより多角的・批判的なインサイトが得られると考えた
- **実現方法案:**
  - LLM: Ollama 等でローカル動作する OSS モデル（例: Llama 3 / Mistral）を2台起動
  - ディベートループ: 一方が「異常の仮説」を提示し、他方が反論・補足するサイクルを繰り返す
  - 入力: 集計ログデータ + フィルタ条件 + 要求定義・要件定義のコンテキスト
  - 出力: ダッシュボード上にインサイトパネルとして表示
- **課題:** LLM のローカル実行にはリソースが必要・レイテンシ管理・ハルシネーション対策

---

## リアルタイム更新（WebSocket）

- **やりたかったこと:** ダッシュボードのデータを WebSocket でリアルタイムに Push する
- **理由:** 運用監視の文脈では、手動更新なしで最新状態を確認したいケースがある
- **実現方法案:** FastAPI WebSocket + フロントエンドの `useWebSocket`
- **今回の代替:** 手動の更新ボタンを設置し、ユーザーが任意のタイミングでデータを再取得できるようにした
