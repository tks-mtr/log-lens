# ドキュメント構成

```
docs/
├── doc_structure.md              ← 本ファイル：ドキュメント構成
├── requirements/                 ← 上流ドキュメント（要求・要件定義）
│   ├── requirements_plan.md          ← 要求・要件定義プラン（feature/requirements-docs）
│   ├── personas_usecases.md          ← ペルソナ・ユースケース定義
│   ├── business_requirements.md      ← 要求定義（What・Why）
│   └── requirements_specification.md ← 要件定義（How）
├── design/                       ← 設計資料・意思決定根拠
│   ├── system_design_plan.md     ← システム設計プラン（feature/system-design）
│   └── tech_selection.md         ← 技術選定（比較・根拠資料）
├── system/                       ← システム設計ドキュメント
│   ├── screen_flow.md            ← 画面遷移図
│   ├── er_diagram.md             ← ER図
│   ├── api_spec.md               ← API仕様書
│   └── sequence.md               ← シーケンス図
├── features/                     ← 機能設計書（実装フェーズで使用）
│   └── YYYYMMDD_機能名/
├── backlog.md                    ← やりたかったこと・将来の改善案
└── summary.md                    ← 意思決定ログ（公開用）
```
