# ドキュメントのルール

## 作成・更新のルール

- ドキュメントを作成・更新した際は、**最低2回**見直しを行うこと
- 1回目: 内容の正確性（事実誤認・技術的な誤り・曖昧な記述）を確認
- 2回目: 表現・構成・読みやすさを確認
- 作業の進捗・予定などの作業管理メモは各フェーズの `_plan.md`（例: `requirements_plan.md`）に記載する
- 要求定義・要件定義・設計ドキュメントなどの成果物には作業管理メモを書かない

## 資料間の整合性チェック

ドキュメントを作成・更新した際は、以下の関連資料との整合性を確認すること。

| チェック対象 | 確認ポイント |
|------------|------------|
| `docs/doc_structure.md` | ファイル名・パスの記載が最終形として正しいか（実態との乖離は許容、_plan.md で管理） |
| 各フェーズの `_plan.md` | 成果物リストに過不足がないか（要求定義・設計・実装フェーズすべて対象） |
| ドキュメント内の相対パスリンク | 参照先ファイル名・パスが正しいか |
| 上位ドキュメントへの参照 | 下位ドキュメントが上位の定義（ペルソナ・機能要求等）と矛盾していないか |

## READMEのルール

- `README.md` は**英語**をデフォルトとする
- `README.ja.md` は日本語版として並行管理する
- 内容を変更した際は**両ファイルを必ず更新**すること
- `README.md` の先頭に `> [日本語版はこちら](README.ja.md)` のリンクを維持すること
- コミットタイミングは `rules_git.md` の「PR直前コミットのルール」を参照
